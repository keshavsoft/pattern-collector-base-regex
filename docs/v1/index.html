Pattern Collector Base Regex

A lightweight, high-performance utility to run a single line against a custom regular expression and extract capture groups as structured parameters.

🚀 Zero Dependencies ⚙️ Regex Matching 📦 ESM Native 🔥 Pure JS

📦 Installation

Install the package into your project using npm:

npm install pattern-collector-base-regex

Or using yarn:

yarn add pattern-collector-base-regex

💡 Concept (Raka & Poka)

For any given input matching line, the utility maps the first two regular expression captured groups to standardized properties:

  • variable / poka: First captured group (representing the input variable or alias name).
  • folderName / raka: Second captured group (representing the output folder name or directory).
The name aliases poka and raka represent the input parameter and output folder respectively, guaranteeing self-contained matching logic from the input string alone.

🛠️ API & Usage Example (v3)

Here is how the default exported function works when called. Pass an options object containing matchLine, parseRegex, and optional showLog.

import patternCollectorBaseRegex from 'pattern-collector-base-regex';

// A regex with 2 capture groups:
// 1. (\w+) -> captures variable/alias name
// 2. ([^/]+) -> captures first directory level
const parseRegex = /import\s*\{[^}]*router\s+as\s+(\w+)[^}]*\}\s*from\s*['"]\.\/([^/]+)\/.*['"]/;

const result = patternCollectorBaseRegex({
    matchLine: "import { router as routerFromv1 } from './v1/routes.js'",
    parseRegex,
    showLog: true
});

console.log(result);
/*
Output:
{
  variable: 'routerFromv1',
  folderName: 'v1',
  raka: 'v1',
  poka: 'routerFromv1'
}
*/
